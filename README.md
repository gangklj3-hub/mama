# ChunkWar (청크전쟁) 플러그인

기획서에 맞춰 만든 Spigot/Paper 플러그인입니다. Maven 프로젝트로 되어있습니다.

> ⚠️ **대상 버전: Minecraft 26.1.2** (2026년부터 마인크래프트가 연도 기반 버전 체계로 바뀌었습니다. 26.1이 2026년 첫 정식 업데이트, 26.1.2는 그 두번째 핫픽스입니다.)
> 이 버전은 이전 버전들과 비교했을 때 변화가 커요:
> - **Java 25 필수** (Java 17/21로는 서버가 아예 안 켜집니다)
> - 서버 jar가 더 이상 난독화되지 않은 형태로 배포됩니다
> - 인챈트/능력치 관련 API 이름이 바뀌었습니다 (`EFFICIENCY`, `UNBREAKING`, `Attribute.MAX_HEALTH` 등 — 아래 코드에 이미 반영해뒀습니다)
> - Spigot용 API는 공개 저장소에 없어서, BuildTools로 로컬에 한 번 빌드해둬야 합니다 (아래 4번 참고)

## 1. 기능 요약

> 참고: 명령어는 `/gamestart`, `/team ...` 처럼 영문으로 되어있습니다 (일부 환경/런처에서 한글 명령어가 인식되지 않는 문제가 있어 영문으로 변경했습니다). 안내 메시지, 팀 이름 등은 그대로 한글 사용 가능합니다.

| 기능 | 명령어 | 설명 |
|---|---|---|
| 게임 시작 | `/gamestart` (관리자) | 1500x1500블록 내 랜덤 텔레포트, "게임시작!" 타이틀, 평화유지시간(20분) 시작, 시작 아이템 지급 |
| 평화유지시간 | (자동) | 20분간 모든 피해 무효(용암/낙하/몹/PvP 등), 팀 생성/가입 가능 |
| 팀 생성 | `/team setleader <닉네임>` (관리자) → `/team create <팀이름>` | 관리자가 임명한 팀장만 팀을 만들 수 있음 |
| 팀 초대 | `/team invite <닉네임>` | 클릭 가능한 [수락]/[거절] 채팅 메시지 전송 |
| 팀 수락/거절 | `/team accept <닉네임>` / `/team deny <닉네임>` | 채팅 버튼 클릭시 자동 실행됨. 거절 시 5분간 재초대 불가 |
| 팀킬 방지 | (자동) | 같은 팀원의 직접 공격(근접/화살)은 무효, 용암 등 설치물 피해는 그대로 적용 |
| 사망 후 관전 | (자동, 평화유지시간 종료 후) | 죽으면 관전(Spectator) 모드로 전환되고, 팀 여부와 상관없이 생존해 있는 아무 플레이어에게나 텔레포트됩니다. 이후에는 카메라가 따라다니지 않는 자유 관전입니다. |
| 전멸/탈락 안내 | (자동) | 소속 팀 전원이 죽거나, 팀 없는 플레이어가 죽으면 전체 화면에 "팀이름(또는 닉네임) 탈락!" 타이틀이 표시됩니다. |
| 자기장 | (자동) | 게임시작 25/35/50분 후 1/2/3단계로 순차 축소, 방어구 무시 고정피해 |
| 게임 종료 | (자동) | 마지막 한 팀(또는 무소속 1인)만 생존 시 자동 종료. 승리 팀원에게 발광(Glowing) 이펙트 부여 + 전체 화면에 "팀이름 승리!" 타이틀 표시 |
| 맵 사전 생성 | (자동, `/gamestart` 시) | 1500x1500 블록(약 8,800개 청크) 전체를 백그라운드에서 강제 로드/생성 |

## 2. 프로젝트 구조

```
ChunkWar/
 ├─ pom.xml
 └─ src/main/
     ├─ java/com/chunkwar/
     │   ├─ ChunkWarPlugin.java        # 메인 클래스 (등록/초기화)
     │   ├─ game/GameManager.java      # 게임 시작, 평화유지시간, 보스바, 시작 키트
     │   ├─ team/Team.java             # 팀 데이터
     │   ├─ team/TeamManager.java      # 팀 생성/초대/수락/거절
     │   ├─ border/BorderManager.java  # 자기장 축소 로직
     │   ├─ border/ChunkPreGenerator.java # 게임시작시 맵 범위 청크 강제 사전생성
     │   ├─ commands/GameStartCommand.java
     │   ├─ commands/TeamCommand.java
     │   ├─ listeners/DamageListener.java  # 평화유지시간 무적 + 팀킬 방지
     │   └─ listeners/PlayerStateListener.java # 사망/퇴장 처리 + 승리 판정 트리거
     └─ resources/
         ├─ plugin.yml
         └─ config.yml
```

## 3. 서버 준비 (플러그인 개발/구동에 필요한 기본 환경)

1. **Java 25** 설치 (Minecraft 26.1 버전대부터 Java 25가 필수입니다. 이전 Java 17/21로는 서버 자체가 안 켜집니다)
2. **Paper 서버**(권장) 또는 Spigot 서버 jar 다운로드
   - Paper: https://papermc.io/downloads
   - 처음 실행 후 생성되는 `eula.txt`에서 `eula=true`로 변경
3. 서버를 한 번 실행해 `plugins` 폴더 등 기본 구조 생성
4. 참고: 기본 맵 크기는 1500x1500블록(약 8,800개 청크)으로, 10인 기준으로 적당한 크기입니다. `/gamestart` 실행 시 플러그인이 자동으로 이 범위를 사전 생성해주므로 별도 사전생성 플러그인은 필요 없습니다.

## 4. 개발 환경 (플러그인을 직접 빌드하려는 경우)

1. IntelliJ IDEA(또는 원하는 IDE) + JDK 17
2. Maven이 설치되어 있거나 IDE 내장 Maven 사용
3. 본 프로젝트 폴더를 열고 `pom.xml`을 Maven 프로젝트로 임포트
4. `pom.xml`은 현재 `26.1.2-R0.1-SNAPSHOT` 버전으로 설정되어 있습니다. 다른 버전 서버를 쓰신다면 이 값을 서버 버전에 맞게 수정하세요.
   - **중요**: 26.1.2용 spigot-api는 Maven Central 같은 공개 저장소에 없습니다. 아래처럼 BuildTools를 **반드시 한 번 실행**해서 로컬(`~/.m2`) 저장소에 설치해야 `mvn package`가 성공합니다.
     ```bash
     java -jar BuildTools.jar --rev 26.1.2
     ```
     (BuildTools 자체를 처음 준비하는 방법은 이전에 안내드린 "서버 여는 법" 3단계와 동일합니다.)
   - Paper를 쓰신다면 `io.papermc.paper:paper-api` (버전 예: `[26.1.2.build,)`, 저장소 `https://repo.papermc.io/repository/maven-public/`)로 교체하는 것도 가능합니다. 26.1 버전대부터는 Paper가 Spigot과 완전히 독립된 프로젝트로 갈라져서, 최신 버전일수록 Paper 쪽 문서/커뮤니티 지원이 더 활발합니다.

## 5. 빌드 및 설치

### 방법 A: 로컬에서 직접 빌드 (Java, Maven, BuildTools 설치 필요)

```bash
mvn clean package
```

- 빌드가 끝나면 `target/ChunkWar.jar` 파일이 생성됩니다.
- 이 jar 파일을 서버의 `plugins` 폴더에 넣고 서버를 재시작(또는 `/reload`, 권장하지 않음)하면 적용됩니다.
- 최초 실행 시 `plugins/ChunkWar/config.yml`이 자동 생성됩니다.

### 방법 B: GitHub Actions로 자동 빌드 (내 컴퓨터에 아무것도 안 깔아도 됨)

이 프로젝트에는 `.github/workflows/build.yml` 이 이미 포함되어 있습니다. GitHub에 올리기만 하면 GitHub의 서버가 대신 Java 25 설치 → BuildTools로 spigot-api 26.1.2 빌드 → `mvn package` 를 자동으로 실행해줍니다.

**사용 방법:**
1. https://github.com 에서 무료 계정 가입 후 로그인
2. 오른쪽 위 `+` → `New repository` 로 새 저장소 생성 (이름은 아무거나, 예: `ChunkWar`)
3. 압축 풀어둔 `ChunkWar` 폴더의 내용물 전체(하위 폴더 포함)를 그 저장소에 업로드
   - 저장소 페이지에서 `Add file` → `Upload files` 로 드래그 앤 드롭 가능
   - `.github` 폴더처럼 점(`.`)으로 시작하는 폴더도 반드시 같이 올라가야 합니다 (탐색기에서 숨김 폴더가 안 보이면 "숨긴 항목 보기"를 켜주세요)
4. 업로드 후 저장소 상단의 **Actions** 탭 클릭
5. `Build ChunkWar Plugin` 워크플로가 자동으로 실행 중일 거예요 (안 보이면 `Run workflow` 버튼으로 수동 실행). **5~15분 정도** 걸립니다 (Spigot을 처음부터 빌드하는 과정이라 시간이 좀 걸려요)
6. 초록색 체크(✅)로 완료되면 그 실행 결과 페이지 맨 아래 **Artifacts** 항목에서 `ChunkWar-plugin`을 다운로드
7. 압축을 풀면 `ChunkWar.jar`가 나옵니다 → 이걸 서버 `plugins` 폴더에 넣으면 끝

> 참고: Public(공개) 저장소면 완전 무료입니다. Private(비공개)로 해도 매달 무료 사용량이 넉넉해서 이 정도 빌드 몇 번으로는 비용이 들지 않아요.

## 6. config.yml 주요 설정값

- `world-name`: 게임이 진행될 월드 이름
- `initial-border-size-blocks`: 초기 플레이 구역(블록, 기본 1500) — 한 변 길이입니다
- `peace-time-seconds`: 평화유지시간(초, 기본 1200 = 20분)
- `border-stage-1/2/3`: 각 단계 발동 시각(게임 시작 후 경과 초), 축소량(블록), 고정 피해량
- `invite-reject-cooldown-seconds`: 팀 초대 거절 후 재초대 대기시간(초, 기본 300 = 5분)
- `pregen-on-start`: `/gamestart` 시 맵 범위 청크 전체를 강제로 미리 생성할지 여부 (기본 true)
- `pregen-chunks-per-tick`: 틱당 몇 개의 청크를 생성할지 (기본 20). 숫자를 올리면 더 빨리 끝나지만 렉이 심해집니다.

## 7. 기획서에서 모호했던 부분 — 이렇게 가정하고 구현했습니다

기획서만으로 100% 명확하지 않았던 부분들이 있어, 합리적으로 판단해 아래와 같이 구현했습니다. 원하시는 방향과 다르면 말씀해주세요, 바로 수정해드립니다.

1. **팀장 지정 방식**: "서버장이 팀장을 정해준다"를 `/team setleader <닉네임>` 명령어로 구현했습니다. 이 명령어를 실행해야 해당 플레이어가 `/team create`를 쓸 수 있습니다.
2. **자기장 피해 주기**: "매 초"라는 명시가 없어 `border-damage-interval-seconds`(기본 1초)마다 피해를 주도록 설정했습니다. config에서 조절 가능합니다.
3. **자기장 중심 좌표**: 기본값은 (0,0) 블록이며 config에서 변경 가능합니다.
4. **게임 종료/승리 조건**: 마지막으로 남은 한 팀(또는 팀이 없는 마지막 1인)이 남으면 자동으로 승리 처리됩니다. (자세한 내용은 7번 참고)
5. **팀 생성/가입이 평화유지시간에만 가능한지**: 기획서 문장이 "평화유지시간에 팀을 만들 수도, 들어갈 수도 있다"였는데, 이를 "평화유지시간 이후에는 불가능하다"는 제약으로 보기보다는 "평화유지시간에도 가능하다"는 허용 문구로 해석해, 현재는 게임 진행 중 언제든 팀 생성/초대가 가능하도록 구현했습니다. 평화유지시간에만 팀을 만들 수 있도록 제한하고 싶다면 쉽게 수정 가능합니다.
6. **인챈트/능력치 API 이름**: 26.1.2 기준으로 `Enchantment.EFFICIENCY`(효율), `Enchantment.UNBREAKING`(내구성), `Attribute.MAX_HEALTH`(최대체력)를 사용하도록 맞춰뒀습니다. 이전 버전(1.20.4 이하)의 `DIG_SPEED`/`DURABILITY`/`GENERIC_MAX_HEALTH` 같은 옛 이름은 더 이상 쓰지 않습니다.
7. **게임 종료 판정 방식**: 참가한 팀(또는 무소속 개인)이 한 팀만 남으면 즉시 승리로 처리합니다. 사망(`PlayerDeathEvent`)뿐 아니라 **게임 도중 서버를 나가는 것도 탈락으로 처리**했습니다(그렇지 않으면 접속을 끊은 마지막 팀원 때문에 게임이 영원히 안 끝날 수 있어서입니다).
8. **사망 후 관전 처리**: 평화유지시간이 끝난 뒤 죽으면 부활 시 자동으로 관전(Spectator) 모드가 되고, 팀 여부와 관계없이 현재 생존한 플레이어 중 아무나 무작위로 골라 그 위치로 텔레포트합니다. 그 이후 카메라가 대상을 계속 따라다니지는 않고(자유 관전), 직접 날아다니며 구경하면 됩니다. 팀장이 죽는 것은 특별 취급하지 않고 일반적인 사망과 동일하게 처리됩니다(팀은 리더가 죽어도 다른 팀원이 살아있으면 유지됩니다). 같은 팀 전원이 죽거나(전멸) 팀 없는 플레이어가 죽으면, 그 즉시 전체 화면에 "팀이름/닉네임 탈락!" 타이틀이 뜹니다.
9. **발광 이펙트 지속시간**: 승리 시 부여되는 Glowing 효과는 사실상 무제한(최대 tick)으로 설정했습니다. 다음 게임 시작 시 자동으로 제거됩니다.
10. **랜덤 텔레포트 안전성**: 최고 높이 블록 위로 텔레포트하되, 용암 위/바닷속 등 세밀한 안전 검사는 하지 않았습니다(단순 "발밑에 블록이 있는지"만 확인). 필요하시면 더 정교한 안전 지점 탐색 로직을 추가해드릴 수 있습니다.

## 8. 확장하면 좋은 것들 (다음 단계 제안)

- 팀별 스코어보드(우측 사이드바)로 남은 팀원 수, 생존 팀 표시
- `/team list`, `/team leave` 등 편의 명령어
- 자기장 경계선을 파티클로 시각화
